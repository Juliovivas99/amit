Terms

Estimate Assumptions

  • Client provides access to necessary tools, accounts, and APIs within agreed timeframes
  • Some platforms may have API limitations requiring exports, workarounds, or phased integration
  • WhatsApp, Teams, Yardi, CoStar, Entrata, PropertyShark, and similar systems may require API approval or vendor support
  • AI features require structured data and approved access to be effective
  • Timeline depends on client responsiveness and scope decisions
  • Initial version prioritizes highest-value workflows first
  • Team of approximately five users; deal volume of 2–3 per month
  • No native mobile application in initial scope
  • Historical data import limited unless specifically scoped

Estimate Validity

Valid for 60 days from June 22, 2026. This proposal is intended to define the initial direction, scope, and pricing structure for the project. Final implementation details may be adjusted based on discovery, technical feasibility, and client priorities. See the full Client Proposal for engagement terms and recommendation.
